# 공항 POS 시스템 화면설계서 초안

## 1. 문서 개요

- 기준 문서: `docs/planning/요구사항정의_pos-system.md`
- 문서 목적: 요구사항 정의 내용을 바탕으로 관리자용 웹 화면 구조와 주요 UI 요소를 화면 단위로 정리한다.
- 설계 범위: MVP 기준 핵심 운영 화면과 필수 모달을 우선 정의한다.
- 설계 원칙:
  - 화면 구조와 업무 흐름을 먼저 고정하고, 시각 스타일은 이후 UI 시안 단계에서 구체화한다.
  - 이상 판정 결과만 보여주지 않고 판단 근거와 비교 데이터를 함께 보여준다.
  - `대기 / 정상 / 확인 필요 / 이상 의심` 상태를 운영자가 한눈에 구분할 수 있어야 한다.

## 2. 설계 범위와 전제

- 사용자 범위:
  - 공항 운영 담당자
  - 감사/정산 담당자
  - 시스템 운영자
- 디바이스 범위:
  - 데스크톱 우선
  - 태블릿 대응
  - 모바일은 조회 중심 반응형 대응
- 인증 전제:
  - 관리자 로그인 화면이 필요하다.
  - 인증 방식은 `SSO 우선 검토`, 필요 시 일반 로그인 보조 방식을 고려한다.
- 데이터 전제:
  - 비교 기준 금액은 `총매출` 또는 `순매출` 중 운영 정책에 따라 달라질 수 있다.
  - 외부 데이터는 이상 판정 보정값이 아니라 매출 변동 해석용 참고 정보로 취급한다.

## 3. 화면 목록

| 화면 ID | 화면명 | 경로 | 목적 | 우선순위 |
| --- | --- | --- | --- | --- |

## 4. 사용자 흐름

1. 사용자는 로그인 화면에서 인증 후 메인 대시보드로 진입한다.
2. 메인 대시보드에서 기준 월, 상태, 매장을 조회하고 위험도 순위를 확인한다.
3. 사용자는 특정 매장을 선택해 매장 상세 화면으로 이동한다.
4. 공항 본사 담당자는 필요 시 수기 등록 또는 엑셀 업로드로 매출신고 데이터를 접수한다.
5. 시스템은 필수 데이터 접수 상태에 따라 `대기` 또는 비교 완료 상태를 표시한다.
6. 사용자는 비교 결과와 근거를 확인한 뒤 후속 검토를 수행한다.

## 5. 공통 구조 및 네비게이션

### 5.1 공통 레이아웃

- 데스크톱:
  - 좌측 사이드바 네비게이션
  - 상단 글로벌 헤더
  - 본문은 화면 목적에 따라 카드, 테이블, 그래프 중심 구성
- 태블릿/모바일:
  - 사이드바는 햄버거 메뉴 또는 드로어로 전환
  - 요약 카드와 필터 영역은 세로 적층
  - 넓은 테이블은 가로 스크롤 또는 카드형 요약으로 축약

### 5.2 글로벌 헤더

- 서비스명
- 현재 접속 사용자 정보
- 기준 월 표시
- 로그아웃

### 5.3 좌측 네비게이션

- `메인 대시보드`
- `매출신고 수기 등록`
- `엑셀 업로드`

### 5.4 공통 상태 배지

| 상태 | 의미 | 기본 색상 |
| --- | --- | --- |
| `대기` | 필수 데이터 미접수 또는 비교 미실행 | 회색 |
| `정상` | 허용 범위 내 정합 | 초록 |
| `확인 필요` | 차이는 있으나 설명 가능성 존재 | 노랑 |
| `이상 의심` | 유의미한 차이 또는 반복 불일치 | 빨강 |

추가로 내부 처리 상태는 별도 보조 라벨로 표시한다.

- `접수완료`
- `대기`
- `비교완료`
- `재비교필요`

## 6. 화면별 상세 설계

### 6.1 `LGN-01` 로그인

#### 목적

- 관리자 인증 후 내부 운영 화면 접근을 제어한다.

#### 진입 조건

- 비로그인 상태에서 시스템 최초 접근
- 세션 만료 후 재접근

#### 주요 구성

- 좌측 또는 상단 브랜드 영역
  - 서비스명
  - 한 줄 설명: 공항 입점 매장 매출 이상 감지 관리자 시스템
- 로그인 패널
  - `SSO 로그인` 기본 버튼
  - 선택형 일반 로그인 영역
  - 아이디
  - 비밀번호
  - 로그인 버튼
- 안내 문구 영역
  - 접근 대상 안내
  - 권한 문의 안내

#### 상태

- 기본 상태
- 로그인 처리 중
- 인증 실패
- 세션 만료 안내

#### 인터랙션

- 인증 성공 시 `메인 대시보드`로 이동
- 인증 실패 시 오류 문구 노출 후 재입력 가능
- 일반 로그인이 비활성 정책이면 입력 필드는 숨기고 SSO 버튼만 노출

### 6.2 `DSH-01` 메인 대시보드

#### 목적

- 기준 월의 매장별 이상 감지 결과를 상태와 위험도 순위 중심으로 빠르게 확인한다.

#### 진입 조건

- 로그인 성공 후 기본 랜딩 화면

#### 화면 구성

1. 상단 페이지 헤더
   - 화면명
   - 기준 월 선택
   - 데이터 접수 및 비교 기준 요약 문구
   - 우측 액션 버튼
   - `수기 등록`
   - `엑셀 업로드`

2. 요약 카드 영역
   - 전체 매장 수
   - `대기` 매장 수
   - `정상` 매장 수
   - `확인 필요` 매장 수
   - `이상 의심` 매장 수

3. 필터 및 검색 영역
   - 상태 필터
   - 매장명/매장코드 검색
   - 노출 여부 필터
   - 정렬 기준
   - 기본값은 위험도 높은 순

4. 위험도 순위 테이블
   - 컬럼 예시:
   - 순위
   - 상태 배지
   - 매장명
   - POS 매장코드
   - 신고 기준
   - 신고금액
   - 비교 기준값
   - 차이금액
   - 차이율
   - 전자결제 정합성 요약
   - 내부 처리 상태
   - 최종 갱신일시
   - `상세 보기`

5. 보조 정보 영역
   - 월간 비교 실행 기준 안내
   - 누락 데이터 경고
   - 최근 재비교 발생 건 요약

#### 핵심 동작

- 테이블 행 클릭 시 `매장 상세`로 이동
- 상태 배지는 색상과 텍스트를 동시에 표시
- `대기` 행은 누락 데이터 항목을 태그 형태로 노출
- 정렬 기본값은 위험도 우선순위이며 사용자가 변경 가능

#### 상태

- 초기 로딩
- 데이터 없음
- 일부 매장만 대기 상태
- 전체 비교 완료
- 오류

#### 비고

- 별도 `매장 등록` 메뉴는 두지 않는다.
- 매장은 POS 기준으로 자동 생성되며, 대시보드에서는 노출/숨김만 제어한다.

### 6.3 `STD-01` 매장 상세

#### 목적

- 특정 매장의 금월 비교 결과, 추이, 판단 근거를 상세하게 확인한다.

#### 진입 조건

- 메인 대시보드에서 특정 매장을 선택

#### 화면 구성

1. 상단 헤더
   - 뒤로가기
   - 매장명
   - POS 매장코드
   - 현재 상태 배지
   - 기준 월

2. 핵심 비교 요약 카드
   - 별도 상단 요약 카드는 두지 않는다.
   - 상세 첫 영역에서 바로 `비교 대상 금액 상세`를 보여주고, `총 POS 금액`, `VAN`, `PG`, `현금`, `할인`, `취소/환불`을 한 번에 노출한다.
   - 같은 영역에서 `신고금액`, `비교 기준값`, `차이금액`, `차이율`, `최종 판정`을 결과 카드로 함께 보여준다.
   - `최종 판정`도 다른 결과 카드와 같은 시각적 무게의 카드로 유지한다.
   - `최종 판정`은 pill 배지 대신 큰 텍스트 중심 표현을 우선 사용한다.
   - 비교 공식이나 복잡한 해석 지표는 MVP 화면에서 기본 노출하지 않고, 먼저 금액 자체를 직관적으로 나열하는 방식으로 정리한다.
   - `VAN`, `PG`, `현금`처럼 더해지는 항목은 `+금액`, `할인`, `취소/환불`처럼 차감되는 항목은 `-금액`으로 직접 표기하고, 색상으로도 함께 구분한다.
   - 비교 기준값이 `총매출`인지 `순매출`인지 `비교 대상 금액 상세` 보조 문구로 명확히 표시한다.

3. 판단 근거 패널
   - 상태 판정 사유 요약
   - 확인된 불일치 포인트
   - 예외 가능성 요약
   - 예시:
   - 할인 급증
   - 현금 비중 급증
   - 자정 경계 거래 다수

4. 운영 이력
   - `상태 전이 이력`
   - `확인 이력`
   - 각 항목은 발생 시각, 주체, 요약 내용을 보여준다.
   - 판단 근거 패널 바로 아래에 배치해 현재 판단과 과거 조치 맥락을 이어서 볼 수 있게 한다.
   - 현재 단계에서는 관리자가 매장을 확인한 기록만 로그 형태로 노출한다.

5. 월별 매출 추이 그래프
   - 최근 12개월 기준
   - 금월과 전월 비교 강조
   - 필요 시 `신고금액`, `비교 기준값` 2개 시리즈 표시

6. 일별 매출 캘린더
   - 일자별 `신고금액`, `POS`를 우선 표기한다.
   - `VAN/PG`는 기본 셀에서 제외하고 필요 시 선택 일자 상세나 별도 상세 영역에서 확인한다.
   - `신고금액`과 `POS`는 서로 다른 컬러 바 또는 시각 표식으로 한눈에 구분되게 한다.
   - 캘린더 상단 보조 영역에 각 색상 또는 컬러 바 의미를 설명하는 범례를 노출한다.
   - 캘린더 상단 보조 영역에 `회색 / 초록 / 노랑 / 빨강` 상태 테두리 의미도 함께 노출한다.
   - 상태 범례에 마우스를 올리면 해당 단계의 차이율 기준값을 툴팁으로 확인할 수 있어야 한다.
   - 개별 날짜 셀에는 반복 라벨을 최소화하고, 금액 왼쪽 컬러 바로 항목을 구분한다.
   - 주간 흐름을 빠르게 읽을 수 있도록 상단에 `월~일` 7열 요일 헤더를 고정 노출한다.
   - 날짜 셀은 동일한 7열 그리드를 사용하고, 월 시작 요일 이전 구간은 빈 셀로 채워 정렬을 맞춘다.
   - 차이 수준은 셀 내부 문구나 아이콘이 아니라 `테두리 색상`으로만 구분한다.
   - 테두리 색상 기준 예시:
   - `회색`: 비교 미실행 또는 유의미한 차이 없음
   - `초록`: 정상 범위
   - `노랑`: 확인 필요
   - `빨강`: 이상 의심
   - 임시 기준 예시:
   - `회색`: 비교 불가 / 데이터 없음
   - `초록`: `차이율 3% 이하`
   - `노랑`: `차이율 3% 초과 10% 이하`
   - `빨강`: `차이율 10% 초과`
   - 셀 선택 시 하단에 해당 일자의 요약 상세를 표시

7. 대사 이슈 목록
   - 일자
   - 이슈 유형
   - 영향 금액
   - 설명 가능성
   - 상태

8. 외부 참고 데이터 영역
   - 탑승객 수
   - 날씨
   - 항공편 운항 정보
   - 매장 상태 변화
   - 외부 데이터가 없는 경우 영역을 숨기지 않고 `미연동` 상태로 표시

#### 핵심 동작

- 월 선택은 가능하되 기본 비교축은 항상 `전월 대비`
- 캘린더 날짜 선택 시 하단 이슈 목록이 해당 일자 기준으로 필터링
- 상태 배지와 판단 근거는 항상 상단 접힘 없이 노출

#### 상태

- 정상 매장 상세
- 확인 필요 매장 상세
- 이상 의심 매장 상세
- 대기 매장 상세
- 외부 데이터 미연동
- 그래프/캘린더 데이터 없음

### 6.4 `RPT-01` 매출신고 수기 등록

#### 목적

- 단건 신고 접수, 정정 입력, 누락 보완을 처리한다.

#### 진입 조건

- 메인 대시보드 상단 액션 또는 좌측 메뉴

#### 화면 구성

1. 상단 헤더
   - 화면명
   - 저장 후 이동 경로 안내

2. 입력 폼
   - 신고 대상 월
   - POS 매장코드
   - 매장명 자동 표시 또는 조회 결과 표시
   - 신고 기준 선택
   - `총매출`
   - `순매출`
   - 신고금액
   - 정정 여부
   - 접수 메모

3. 우측 또는 하단 보조 안내
   - 동일 월/동일 매장 중복 접수 규칙
   - 비교 실행 조건
   - 저장 후 예상 상태 안내

4. 하단 액션
   - 임시 취소
   - 저장

#### 검증 규칙

- 신고 대상 월 필수
- POS 매장코드 필수
- 신고 기준 필수
- 신고금액은 0보다 큰 숫자
- 동일 월/동일 매장 데이터가 이미 존재하면 `정정 등록` 여부 확인

#### 저장 결과

- 필수 비교 데이터가 부족하면 `대기`
- 비교 가능 조건을 만족하면 저장 후 비교 대기 또는 재비교 대상 표시

#### 상태

- 기본 입력
- 필수값 누락
- 중복 접수 경고
- 저장 완료
- 저장 실패

### 6.5 `RPT-02` 매출신고 엑셀 업로드

#### 목적

- 월말 다수 매장의 신고 내역을 일괄 접수한다.

#### 진입 조건

- 메인 대시보드 상단 액션 또는 좌측 메뉴

#### 화면 구성

1. 상단 헤더
   - 화면명
   - 템플릿 사용 안내

2. 업로드 가이드 영역
   - 템플릿 다운로드 버튼
   - 필수 컬럼 안내
   - 허용 파일 형식 안내

3. 파일 업로드 영역
   - 드래그 앤 드롭
   - 파일 선택 버튼

4. 검증 결과 미리보기
   - 총 행 수
   - 정상 행 수
   - 오류 행 수
   - 경고 행 수
   - 행별 오류 테이블

5. 하단 액션
   - 업로드 취소
   - 검증 다시 실행
   - 접수 확정

#### 검증 규칙

- 필수 컬럼 존재 여부 확인
- 월, 매장코드, 신고금액, 신고 기준 형식 확인
- 동일 월/동일 매장 중복 여부 확인
- 숫자 형식 오류 확인
- 미확인 POS 매장코드는 즉시 차단하지 않고 경고 대상으로 표시할지 정책 확인 필요

#### 결과 처리

- 정상 행만 접수
- 오류 행은 접수 제외 후 상세 사유 표시
- 접수 후 `업로드 검증 결과` 모달 노출

#### 상태

- 파일 미선택
- 업로드 중
- 검증 완료
- 일부 오류 포함
- 접수 완료
- 접수 실패

### 6.6 `POP-01` 운영 설정

#### 목적

- 대시보드에 표시할 매장과 알림 수신 방식을 운영자가 제어한다.

#### 호출 위치

- 메인 대시보드의 `톱니바퀴` 설정 버튼

#### 주요 구성

- 섹션 구분
- 매장 검색
- 매장명
- POS 매장코드
- 현재 노출 여부 토글
- `PC 알림 수신` 토글
- 브라우저 권한 상태 표시
- `권한 요청` 버튼
- `테스트 알림 보내기` 버튼
- 저장 버튼

#### 동작 규칙

- 매장 신규 생성은 하지 않는다.
- 노출/숨김 설정만 허용한다.
- 숨김 처리된 매장은 기본 대시보드 목록에서 제외하되 필요 시 필터로 조회 가능하게 한다.
- `PC 알림 수신`을 켠 경우에도 브라우저 권한이 허용되지 않으면 시스템 알림은 발송되지 않는다.
- `테스트 알림 보내기`는 실제 서버 수신 이벤트 대신 프로토타입 알림 흐름을 검증하는 용도로 사용한다.
- 모달 내용이 화면 높이를 초과하는 경우 모달 내부에서 스크롤되도록 처리한다.
- 인앱 토스트는 등장 시 아래에서 위로 부드럽게 올라오며 즉시 주목할 수 있게 처리한다.

### 6.7 `POP-02` 업로드 검증 결과

#### 목적

- 업로드 직후 처리 결과와 후속 조치 필요 항목을 빠르게 확인한다.

#### 호출 위치

- 엑셀 업로드 접수 완료 후

#### 주요 구성

- 접수 성공 건수
- 오류 건수
- 경고 건수
- 오류 다운로드 버튼
- 닫기 버튼
- `대시보드로 이동`

## 7. 공통 입력 및 검증 규칙

- 기준 월 관련 입력은 `YYYY-MM` 단위로 통일한다.
- 금액 입력은 천 단위 구분 표시를 하되 내부 저장값은 숫자로 처리한다.
- `총매출/순매출` 선택은 라디오 버튼으로 단일 선택한다.
- 동일 월/동일 매장 중복 건은 신규 저장 전에 반드시 경고한다.
- 정정 등록 시 이전 이력이 존재함을 화면에 명시한다.

## 8. 오류 및 예외 처리

- 로그인 실패 시 원인 노출은 최소화하고 재시도 경로를 제공한다.
- 대시보드 조회 실패 시 전체 오류 화면 대신 재조회 버튼을 제공한다.
- `대기` 상태는 오류로 표현하지 않고 데이터 미완료 상태로 표현한다.
- 외부 데이터가 없을 때는 기능 실패가 아니라 `미연동` 또는 `참고 데이터 없음`으로 표기한다.
- 업로드 오류는 파일 전체 실패와 일부 행 실패를 구분해 안내한다.

## 10. 반응형 및 사용성 고려사항

- 데스크톱에서는 표와 그래프를 동시에 보여주되, 태블릿 이하에서는 카드와 차트 중심으로 재배치한다.
- 위험도와 상태는 텍스트 없이 색만으로 표현하지 않는다.
- 숫자 정보는 `금액`, `차이금액`, `차이율`이 한 번에 읽히도록 묶어 배치한다.
- 상세 화면에서는 판단 근거 패널을 차트보다 상단에 둔다.
- 대시보드에서는 핵심 액션인 `수기 등록`, `엑셀 업로드`를 항상 첫 화면에서 찾을 수 있어야 한다.
