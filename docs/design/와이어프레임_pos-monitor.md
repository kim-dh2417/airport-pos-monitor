# 공항 POS 시스템 와이어프레임 초안

## 1. 문서 개요

- 기준 문서:
  - `docs/design/화면설계서_pos-monitor.md`
  - `docs/planning/요구사항정의_pos-system.md`
- 대상 화면:
  - `DSH-01 메인 대시보드`
  - `STD-01 매장 상세`
- 목적:
  - 화면 구조, 정보 우선순위, 주요 컴포넌트 배치를 텍스트 기반 와이어프레임으로 정의한다.

## 2. 설계 원칙

- 운영자가 첫 화면에서 `위험도`, `상태`, `차이 규모`를 바로 읽을 수 있어야 한다.
- 상세 화면에서는 그래프보다 `판단 근거`와 `핵심 차이값`이 먼저 보여야 한다.
- 색이 없어도 정보 구조를 이해할 수 있게 라벨, 제목, 수치 위계를 분명히 둔다.
- 데스크톱 기준으로 설계하되 태블릿/모바일 재배치 기준도 함께 정리한다.

## 4. `DSH-01` 메인 대시보드 와이어프레임

### 4.1 화면 목적

- 기준 월의 매장별 위험도와 상태를 한눈에 파악한다.
- 수기 등록과 엑셀 업로드로 바로 이어지는 운영 허브 역할을 한다.

### 4.2 데스크톱 구조

```text
+------------------+------------------------------------------------------------------+
| Sidebar          | Header: 메인 대시보드                            [수기 등록][업로드] |
|                  | 기준 월: 2026-03 | 월간 비교 상태: 완료 | 누락 데이터: 3건         |
|                  +------------------------------------------------------------------+
|                  | [전체 매장 48] [대기 4] [정상 30] [확인 필요 10] [이상 의심 4]     |
|                  +------------------------------------------------------------------+
|                  | 상태 필터 | 검색 | 노출 여부 | 정렬: 위험도 높은 순           |
|                  +------------------------------------------------------------------+
|                  | 위험도 순위 테이블                                               |
|                  |------------------------------------------------------------------|
|                  | 순위 | 상태 | 매장명 | 코드 | 신고기준 | 차이금액 | 차이율 |      |
|                  |      |      |        |      |         | 전자결제 정합성 | 상태   |
|                  |------------------------------------------------------------------|
|                  | 1    | 이상 의심 | A매장   | S01  | 순매출  | -12,500,000 | -18.2% |
|                  | 2    | 확인 필요 | B매장   | S07  | 총매출  | -3,400,000  | -6.1%  |
|                  | ...                                                        [상세]|
|                  +------------------------------------------------------------------+
|                  | 하단 보조 영역                                                  |
|                  | - 재비교 필요 2건                                              |
|                  | - 누락 데이터 경고                                             |
|                  | - 최근 상태 변경 이력 요약                                     |
+------------------+------------------------------------------------------------------+
```

### 4.3 구성 설명

#### 상단 헤더

- 화면명
- 기준 월 선택
- 현재 월 비교 실행 상태
- 우측 고정 액션 버튼:
  - `수기 등록`
  - `엑셀 업로드`

#### 요약 카드

- `전체 매장`
- `대기`
- `정상`
- `확인 필요`
- `이상 의심`

각 카드는 수치 중심으로 보여주고, 클릭 시 해당 상태 필터가 적용되는 구조를 권장한다.

#### 필터 바

- 상태 선택
- 매장명/코드 검색
- 노출 여부 선택
- 정렬 기준

필터는 한 줄 고정 배치하고, 모바일에서는 접기형 필터 패널로 전환한다.

#### 위험도 순위 테이블

- 첫 컬럼은 순위
- 두 번째 컬럼은 상태 배지
- 차이금액과 차이율은 우측 정렬
- `대기` 상태는 누락 데이터 태그를 함께 노출
- `상세` 버튼 없이 행 전체 클릭 이동도 가능하게 설계

#### 하단 보조 영역

- 전체 화면 하단 우측 또는 하단 카드로 배치
- 운영자가 놓치기 쉬운 누락/재비교 정보를 요약한다.

### 4.4 모바일 재배치

```text
+--------------------------------------------------------------+
| Header: 메인 대시보드                        [+수기][+업로드] |
+--------------------------------------------------------------+
| [전체 48] [대기 4] [정상 30]                                |
| [확인 필요 10] [이상 의심 4]                                |
+--------------------------------------------------------------+
| 필터 열기                                                    |
+--------------------------------------------------------------+
| 카드형 리스트                                                |
| [이상 의심] A매장 / S01                                      |
| 신고기준 순매출 | 차이금액 -12,500,000 | 차이율 -18.2%       |
| 전자결제 정합성 불일치 | 내부상태 재비교필요          [상세] |
+--------------------------------------------------------------+
```

모바일에서는 테이블을 카드 리스트로 변환한다.

### 4.5 운영 설정 모달

- 대시보드 설정 버튼 클릭 시 모달로 연다.
- 설정 버튼은 톱니바퀴 아이콘으로 명확히 표현한다.
- 상단은 `알림 설정`, 하단은 `매장 노출 설정` 섹션으로 나눈다.
- 알림 설정에는 `PC 알림 수신` 토글, 브라우저 권한 상태, `권한 요청`, `테스트 알림 보내기` 버튼을 둔다.
- 테스트 알림 실행 시 운영자는 인앱 토스트와 브라우저 시스템 알림 동작 여부를 함께 확인할 수 있어야 한다.
- 모달 높이가 길어질 경우 화면 바깥으로 넘치지 않고 내부 스크롤로 확인 가능해야 한다.
- 인앱 토스트는 하단에서 스르륵 올라오는 짧은 등장 모션을 사용한다.

## 5. `STD-01` 매장 상세 와이어프레임

### 5.1 화면 목적

- 단일 매장의 금월 비교 결과와 판단 근거를 깊게 본다.
- 운영 검토를 위해 차이값, 추이, 일자별 패턴, 대사 이슈를 한 화면 안에서 연결해 본다.

### 5.2 데스크톱 구조

```text
+------------------+------------------------------------------------------------------+
| Sidebar          | [뒤로가기] A매장 상세                       상태: 이상 의심       |
|                  | POS 코드: S01 | 기준 월: 2026-03 | 비교축: 전월 대비          |
|                  +------------------------------------------------------------------+
|                  | [신고금액] [비교 기준값] [차이금액] [차이율] [전자결제 정합성]    |
|                  +--------------------------------------+---------------------------+
|                  | 판단 근거 패널                         | 보조 운영 정보            |
|                  | - 신고금액 차이 초과                  | - 내부 처리 상태          |
|                  | - 카드 승인 합계 불일치               | - 최근 재비교 일시        |
|                  | - 현금 비중 급증                      | - 신고 기준               |
|                  +------------------------------------------------------------------+
|                  | 운영 이력                                                        |
|                  | [상태 전이 이력] [확인 이력]                                     |
|                  | 03-27 09:10 확인 필요 전환 | 03-27 09:30 김운영 확인 기록       |
|                  +------------------------------------------------------------------+
|                  | 월별 매출 추이 그래프                                           |
|                  | [최근 12개월 신고금액 vs 비교 기준값]                          |
|                  +------------------------------------------------------------------+
|                  | 일별 매출 캘린더                      | 선택 일자 상세            |
|                  | 월 화 수 목 금 토 일                  | POS / VAN-PG / 차이값     |
|                  | [ ][ ][1][2][3][4][5]...            | 주요 이슈 / 메모          |
|                  | 차이 발생 일자 강조                   | 주요 이슈 / 메모          |
|                  +------------------------------------------------------------------+
|                  | 대사 이슈 목록                                                    |
|                  | 일자 | 이슈유형 | 영향금액 | 설명 가능성 | 상태                  |
|                  +------------------------------------------------------------------+
|                  | 외부 참고 데이터                                                 |
|                  | 탑승객 | 날씨 | 항공편 | 매장 상태 변화                          |
+------------------+------------------------------------------------------------------+
```

### 5.3 구성 설명

#### 헤더

- `뒤로가기`
- 매장명
- POS 매장코드
- 현재 상태
- 기준 월
- 비교 기준은 문구로 `전월 대비`를 고정 노출

#### 핵심 비교 카드

- 별도 상단 요약 카드는 제거한다.
- 상세 첫 영역에서 바로 `비교 대상 금액 상세`를 보여주고 `총 POS 금액`, `VAN`, `PG`, `현금`, `할인`, `취소/환불` 내역을 짧은 수치 카드로 보여준다.
- 같은 영역 또는 바로 아래에 `신고금액`, `비교 기준값`, `차이금액`, `차이율`, `최종 판정`을 결과 카드로 묶어 현재 수치와 결론을 바로 읽게 한다.
- `최종 판정`도 축소 배지 대신 일반 결과 카드 크기로 유지한다.
- `최종 판정`은 pill 배지 없이 큰 텍스트 중심으로 표시한다.
- 비교 공식, 해석 지표, 중복 설명은 MVP 화면에서 제거한다.
- `VAN`, `PG`, `현금` 카드는 `+금액`, `할인`, `취소/환불` 카드는 `-금액` 형태로 직접 표기하고, 색상으로 방향을 구분한다.
- `비교 대상 금액 상세` 상단 보조 문구로 `총매출 기준` 또는 `순매출 기준`을 함께 둔다.

#### 판단 근거 패널

- 그래프보다 위에 둔다.
- 운영자가 왜 `정상/확인 필요/이상 의심`인지 먼저 이해하게 만드는 영역이다.
- 구조:
  - 핵심 사유 3개 이내
  - 예외 가능성
  - 참고 검토 포인트

#### 운영 이력

- 판단 근거 패널 바로 아래에 둔다.
- `상태 전이 이력`과 `확인 이력`을 나란히 보여준다.
- 시스템이 상태를 바꾼 흐름과 관리자가 확인한 시점을 한 화면에서 이어서 읽게 한다.
- 각 항목은 발생 시각, 주체, 요약 내용을 짧게 표시한다.
- 현재 단계에서는 복잡한 운영 액션 대신 확인 로그만 노출한다.

#### 월별 매출 추이 그래프

- 최소 12개월 구간
- 금월과 전월은 강조 마커 처리
- 2개 시리즈 기준:
  - 신고금액
  - 비교 기준값

#### 일별 매출 캘린더

- 캘린더 상단에 `월~일` 요일 헤더를 고정 노출한다.
- 요일 헤더와 날짜 셀은 같은 `7열` 기준을 공유해야 한다.
- 월 시작 요일 이전의 빈 구간은 빈 셀로 유지해 실제 달력처럼 읽히게 한다.
- 달력 셀마다 우선 `신고금액`, `POS`를 축약 표기한다.
- `VAN/PG`는 캘린더 셀에서 제외하고 선택 일자 상세에서 확인 가능하게 한다.
- `신고금액`, `POS`는 서로 다른 컬러 바로 구분해 숫자 비교가 빠르게 되도록 한다.
- 캘린더 상단에 각 컬러 바 의미를 설명하는 범례를 둔다.
- 캘린더 상단에 상태 테두리 색 의미를 설명하는 보조 범례도 함께 둔다.
- 상태 범례 항목 hover 시 단계별 임시 기준값을 툴팁으로 노출한다.
- 개별 셀에는 `신고`, `POS` 라벨을 반복 노출하지 않고 금액 왼쪽 컬러 바로만 구분한다.
- 개별 날짜 셀에는 별도 요일 표기를 두지 않고, 상단 `월~일` 헤더로만 주간 흐름을 인지하게 한다.
- 상태 텍스트나 경고 아이콘 없이 `테두리 색상`만으로 차이 수준을 구분한다.
- 색상 단계는 `회색 / 초록 / 노랑 / 빨강`을 사용한다.
- 날짜 클릭 시 우측 또는 하단 상세 패널 연동

#### 대사 이슈 목록

- 거래 단위 상세대사 전체표까지는 넣지 않고, 먼저 이슈 요약 표를 둔다.
- 필요 시 향후 `상세대사 보기` 드로어를 추가할 수 있다.

#### 외부 참고 데이터

- 연동 전이라도 공간을 확보해 둔다.
- 데이터가 없으면 `미연동` 플레이스홀더를 노출한다.

### 5.4 모바일 재배치

```text
+--------------------------------------------------------------+
| [뒤로] A매장                                [이상 의심 배지] |
| POS 코드 S01 | 기준 월 2026-03                                  |
+--------------------------------------------------------------+
| [신고금액] [비교값]                                            |
| [차이금액] [차이율]                                            |
| [전자결제 정합성]                                              |
+--------------------------------------------------------------+
| 판단 근거                                                     |
| - 신고금액 차이 초과                                          |
| - 카드 승인 합계 불일치                                       |
+--------------------------------------------------------------+
| 월별 매출 그래프                                               |
+--------------------------------------------------------------+
| 일별 매출 캘린더                                               |
+--------------------------------------------------------------+
| 대사 이슈 목록                                                 |
+--------------------------------------------------------------+
| 외부 참고 데이터                                               |
+--------------------------------------------------------------+
```

모바일에서는 `판단 근거 -> 요약 수치 -> 그래프 -> 캘린더 -> 이슈 목록` 순으로 읽히게 세로 적층한다.

## 6. 컴포넌트 우선순위

### 필수

- 상태 배지
- 요약 카드
- 위험도 테이블
- 월별 추이 차트
- 일별 캘린더
- 판단 근거 패널

### 있으면 좋은 요소

- 누락 데이터 태그
- 내부 처리 상태 라벨
- 외부 데이터 플레이스홀더 카드

## 7. 화면 간 연결 규칙

- 대시보드 행 클릭 시 매장 상세로 이동한다.
- 수기 등록/엑셀 업로드 완료 후 기본 복귀 위치는 대시보드다.
- 상세 화면에서 기준 월을 바꿔도 비교축 문구는 `전월 대비`를 유지한다.
