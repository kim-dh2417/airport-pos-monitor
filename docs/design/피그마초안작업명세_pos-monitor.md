# 공항 POS 시스템 피그마 초안 작업 명세

## 1. 목적

- Figma MCP 호출 한도 해제 후 한 번에 초안을 반영할 수 있도록 화면 배치와 샘플 데이터를 정리한다.
- 대상 범위는 `DSH-01 메인 대시보드`, `STD-01 매장 상세`, 모바일 축약안 2종이다.

## 2. 파일 운영 방식

### 2.1 Starter 플랜 대응

- 원래 목표 페이지 구조:
  - `00 Cover`

  - `02 Screens`

### 2.2 반영 대상 파일

- Figma 파일:
  - `POS Monitor Draft 2026-03-27`
- URL:
  - `https://www.figma.com/design/Wd2KwP74F1n3iSPlfDePlg`

## 3. 공통 디자인 방향

- 밝은 캔버스 기반 운영형 대시보드
- 상태 표현은 `대기 / 정상 / 확인 필요 / 이상 의심` 4단계 고정
- 대시보드는 위험도와 차이 규모가 먼저 읽히도록 구성
- 상세 화면은 그래프보다 `판단 근거`가 먼저 보이도록 구성

## 5. `02 Screens` 페이지 배치 초안

### 5.1 데스크톱 대시보드

- 프레임명:
  - `Frame/Desktop/Dashboard`
- 크기:
  - `1440 x 1400`
- 내부 구조:
  - `Sidebar`
  - `GlobalHeader`
  - `PageHeader`
  - `SummaryRow`
  - `FilterBar`
  - `TablePanel`
  - `SupportCardsRow`

#### 배치 요소

- 좌측 사이드바:
  - `메인 대시보드`
  - `수기 등록`
  - `엑셀 업로드`
- 상단 헤더:
  - 서비스명
  - 기준 월 `2026-03`
  - 사용자 정보
  - 로그아웃
- 페이지 헤더:
  - 제목 `메인 대시보드`
  - 설명 `기준 월의 매장별 위험도와 상태를 빠르게 파악하는 운영 허브`
  - 우측 버튼 `수기 등록`, `엑셀 업로드`
- 요약 카드:
  - 전체 매장 `48`
  - 대기 `4`
  - 정상 `30`
  - 확인 필요 `10`
  - 이상 의심 `4`
- 필터 바:
  - `매장명 또는 코드 검색`
  - `상태 전체`
  - `노출 여부`
  - `위험도 높은 순`
  - 보조 텍스트 `누락 데이터 3건`
- 테이블 샘플 행:
  - `A매장 / 이상 의심 / 순매출 / -12,500,000 / -18.2%`
  - `B매장 / 확인 필요 / 총매출 / -3,400,000 / -6.1%`
  - `C매장 / 정상 / 총매출 / -210,000 / -0.4%`
  - `D매장 / 대기 / 필수 데이터 미접수`
  - `E매장 / 이상 의심 / 카드 승인 편차`
- 하단 보조 카드:
  - `재비교 필요`
  - `누락 데이터 경고`
  - `최근 상태 변경`

### 5.2 데스크톱 매장 상세

- 프레임명:
  - `Frame/Desktop/StoreDetail`
- 크기:
  - `1440 x 1800`
- 내부 구조:
  - `Sidebar`
  - `GlobalHeader`
  - `PageHeader`
  - `MetricCards`
  - `InsightRow`
  - `TrendChartPanel`
  - `CalendarAndDayDetail`
  - `IssueTablePanel`
  - `ExternalDataPanel`

#### 배치 요소

- 상단:
  - `← 메인 대시보드`
  - 상태 배지 `이상 의심`
  - 제목 `A매장`
  - 메타 `POS 코드 S01 | 기준 월 2026-03 | 비교축 전월 대비`
- 핵심 카드:
  - 신고금액 `54.2M`
  - 비교 기준값 `66.7M`
  - 차이금액 `-12.5M`
  - 차이율 `-18.2%`
  - 전자결제 정합성 `87%`
- 판단 근거:
  - 신고금액 차이율이 임계치 초과
  - 카드 승인 합계와 POS 집계 불일치
  - 현금 비중 급증 + 자정 경계 거래 동시 관찰
- 태그:
  - `할인 급증`
  - `자정 경계 거래`
  - `외부 데이터 미연동`
- 운영 정보 카드:
  - 내부 처리 상태 `재비교 필요`
  - 최근 재비교 시각
  - 신고 기준 `총매출`
- 추이 차트:
  - `신고금액`
  - `비교 기준값`
  - 최근 12개월
  - 금월 강조 마커 포함
- 캘린더:
  - 셀 구성 `P / V / D`
  - 경고일 예시:
    - `6일`
    - `11일`
    - `15일`
    - `21일`
- 선택 일자 상세:
  - `03-21`
  - `POS 11.4M / VAN-PG 12.8M`
  - `차이 -1.4M`
- 이슈 테이블 샘플:
  - 카드 승인 누락
  - 현금 비중 급증
  - 자정 경계 거래 집중
  - POS 집계 누락
- 외부 참고 데이터 카드:
  - 탑승객 수
  - 날씨
  - 항공편 운항
  - 매장 상태 변화
